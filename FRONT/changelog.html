Changelogs of CentrixOS

Notes v0.0.7 (Unstable):

  • The option to have Sway as an alternative desktop environment to Hyprland has been cancelled and the latter is left as the only option.

  • Added a feature to change the CentrixOS theme between the settings offered, the same themes are updated automatically.

  • Changed the wallpaper for the "Purple-Swirl" theme.

  • New theme for CentrixOS called "Blue-Depression".

  • The nomenclature of versions inspired by the zodiac signs is removed.

  • The CentrixOS installation script has been translated into English.

  • A new option has been incorporated into the script to enable Debian Trixie/Backports.

Notes v0.0.6 (Unstable):

  • The option of using Openbox as an alternative environment to Hyprland was reconsidered and it was decided to offer Sway as a secondary desktop environment of the user's choice, which is under development.

  • A new customization theme called "Shadow-Dance" was added for Hyprland with mostly monochromatic colors highlighting white.

Notes v0.0.5 (Unstable):

  • CentrixOS v0.0.5 represents the first version with "Unstable" nomenclature and represents a new stage for the development of the CentrixOS project. This version is designed to revamp the visual experience of the project and add new functionality that the experimental versions did not incorporate.

  • New feature of the CentrixOS installation script to be able to choose between two customization variants for Hyprland (Purple-Swirl and Green-Forest).

  • The usefulness of Calcurse to have a calendar via terminal was removed.

  • Custom ASCII art for fastfetch.

  • We plan to implement in the CentrixOS installation menu the option to install the system with OpenBox as a desktop environment together with Hyprland. Replacing Hyprland in future versions to have OpenBox as the main desktop environment is being considered.

Notes v0.0.4 (Experimental):

  • Chromium replaced Brave Browser as the default web browser.

  • New modifications to the installation script.

  • Codenames were implemented for versions of CentrixOS from v0.0.4 onwards inspired by zodiac signs.

  • A custom Fastfetch configuration and the VERSION command were implemented to display the version of CentrixOS being used.

  • The --app function for the artificial intelligence agents was modified to work with Chromium.

  • The resize_on_border option was enabled by default to be able to resize windows when they are in floating mode.

Notes v0.0.3 (Experimental):

  • Improvements have been made to the installation script.

  • Unified pre-install.sh and install.sh scripts into a single installation script.

  • Changed the names of the internal directories.

  • An update process for the script is in the works.

Notes v0.0.2 (Experimental):

  • Added Brave Browser (Required for certain features).

  • New customizations in Wofi and Waybar.

  • Added Calcurse and btop as terminal applications, and Grim for taking screenshots.

  • We have incorporated new .desktop files within the ~/.local/share/applications directory to be able to run AI models (ChatGPT, Claude Code and Google Gemini) using the --app function offered by Chromium-based browsers.

  • It is recommended to test CentrixOS in virtual machines before installing it on physical hardware.

Notes v0.0.1 (Experimental):

  • CentrixOS is not a standalone GNU/Linux distribution, nor is it based on an existing one., it is only a customization layer.

  • This project is still in the development phase so it is not recommended to blindly trust the installation process, improvements and corrections are being worked on.

  • Its main purpose is to be a customization layer on top of Debian using packages from the official Debian repositories and Debian Trixie Backports.

  • Trixie/Backports repositories need to be configured prior to running the script.

  • The script should NOT be executed as root or using sudo, it is recommended to assign execution permissions using chmod +x.

  • The customization uses zsh as the system shell with Starship customization.

  • All bugs will be fixed with the development of new versions.
